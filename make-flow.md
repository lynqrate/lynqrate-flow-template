### Make 시나리오 전체 흐름 - 감정일기 분석 자동화

1. **[Webhook]**  
   - Tally에서 응답이 들어오면 시작
   - 자유 입력 필드 빈값인지 체크 (감정, 오늘 있었던 일, 감정일기, 인증코드)
   - → 하나라도 비어있으면 실패 반환 or fallback

2. *(아직 미구현)* **[Code Validation]**  
   - 입력된 인증코드 (`uuid_code`)로 `user_passes` 테이블에서 해당 이용권 검색
   - → 유효하지 않으면 에러 반환
   - → 유효하면 `remaining_uses` 확인
   - → 유효하고 남은 회차 있으면 계속 진행

3. **[라벨 추출]**
   - 드롭다운으로 들어온 값은 UUID이므로  
     `map()` + `get()`으로 → "감정정도", "피드백 유형" 라벨 추출
   - 이 값을 그대로 GPT에 넣기 위한 `prompt 구성용 변수`로 저장

4. **[감정 정규화 (GPT 호출)]**
   - 사용자가 입력한 감정 단어를 GPT로 → 표준 감정 단어군으로 변환 시도
   - ex: "기분이 구질구질해" → "슬픔"

5. **[표준 감정 여부 판별 (Router)]**
   - GPT가 반환한 감정이 `standard_emotions` 목록 안에 있는지 확인
     - 있음 → 분류 성공
     - 없음 → 분류 실패

6. **[GPT 피드백 생성 분기]**
   - (1) **표준 감정에 해당**할 경우:
     - situation, journal, standard emotion, 정도, 요청유형을 GPT에 넣어 피드백 생성
   - (2) **표준 감정 외 감정일 경우**:
     - GPT로 감정 의미 해석하고 → 감정 자체 + 상황 + 일기 + 요청유형 기반 피드백 생성

7. *(아직 미구현)* **[DB 저장 로직 예정]**
   - emotion_entries: 사용자 raw 입력, 감정 단어, 표준 감정 ID, GPT 분류 사유, 코드에 연결된 `user_pass_id` 저장
   - emotion_feedbacks: GPT 피드백 결과, 토큰, 모델명, 생성일 저장
   - 이후 `user_passes.remaining_uses` 1 감소

8. *(예정)* **[응답 반환]**
   - 요약 결과 + GPT 피드백을 사용자에게 전송 (메일 or WebView or 화면 출력)
